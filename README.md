SILK POS 系統 - 交接與維護手冊

歡迎接手 SILK POS 系統！這是一套專為 iPad 設計的輕量化銷售管理系統。

這份手冊是特別為沒有程式背景的管理者設計的。本系統的維護核心理念是：「複製程式碼給 AI，告訴它你想改什麼，然後把新程式碼貼回去。」

1. 系統架構概念 (必讀)

這個系統由兩個部分組成，請想像成「店面」與「倉庫」：

店面 (GitHub Pages)：就是您在 iPad 上看到的畫面。

檔案位置：GitHub 上的 index.html。

所有功能的修改（改顏色、加按鈕、修流程）都是改這裡。

倉庫 (Firebase)：雲端資料庫，負責記住所有的商品、訂單和庫存。

資料是獨立安全的。 就算您把 index.html 改壞了，只要 Firebase 設定沒動，客人的訂單資料都不會消失。

2. 如何使用系統 (日常操作)

本系統分為四個主要頁面：

銷售 (Sales)：

點選商品或掃描條碼加入購物車。

標籤篩選：可多選標籤（例如同時選「食品」+「冷凍」）。

商品套組：切換分頁可一鍵加入常用組合（如：大禮包）。

結帳：點擊右側結帳，可選擇日期、人員、付款方式與折扣。

庫存 (Inventory)：

快速進貨：直接點擊商品進行單一入庫。

批次進貨單：將商品加入右側清單，確認後一次性入庫（此操作不會產生金額紀錄）。

數量輸入：點擊數字可直接輸入大額數量。

報表 (History)：

查看所有交易紀錄。

編輯：可事後修改訂單的人員、折扣或補登日期。

匯出：可將資料下載為 Excel 檔。

設定 (Settings)：

新增/修改/刪除商品、人員、付款方式、折扣與標籤。

支援拖曳排序。

3. 如何修改與更新功能 (AI 維護流程)

如果您需要修改功能（例如：想把按鈕變大、想增加一個欄位），請嚴格遵守以下步驟：

步驟 A：取得現有程式碼

登入 GitHub。

進入本專案的儲存庫 (Repository)。

點擊 index.html 檔案。

點擊右上角的 「複製圖示 (Copy raw file)」（兩張紙疊在一起的圖示）。請勿直接用滑鼠反白複製，容易出錯。

步驟 B：與 AI 溝通 (ChatGPT / Claude / Gemini)

將複製的程式碼貼給 AI。

輸入指令：清楚告訴 AI 您要改什麼。

範例：「這是我的 POS 系統代碼，請幫我把『確認結帳』的按鈕改成紅色。」

範例：「請幫我在設定頁面增加一個『會員備註』的欄位。」

接收新代碼：AI 會生成一份新的 index.html。請點擊 AI 對話框代碼區塊右上角的 「Copy」 按鈕。

步驟 C：更新上線

回到 GitHub 的 index.html 頁面。

點擊右上角的 「筆圖示 (Edit this file)」。

關鍵步驟：在編輯區內按 Ctrl + A (全選) -> Delete (刪除)。一定要先清空舊的，不能直接貼上。

按下 Ctrl + V (貼上) AI 給您的新程式碼。

捲動到頁面最下方，點擊綠色的 「Commit changes」 按鈕。

等待約 1-2 分鐘，iPad 重新整理網頁即可看到更新。

4. 圖片上傳教學

系統使用「圖片網址」來顯示商品照。

建議使用 ImgBB (免註冊) 或將圖片上傳至此 GitHub 的 images 資料夾。

取得圖片的 「直接連結 (Direct Link)」（網址結尾必須是 .jpg 或 .png）。

進入系統 「設定」 -> 編輯商品 -> 將網址貼入 「圖片連結」 欄位。

5. 常見問題排除 (Troubleshooting)

狀況 1：更新後網頁變一片白，什麼都沒有？

原因：通常是複製貼上時弄錯了（例如少複製一行，或是沒有先清空舊代碼）。

解法：去 GitHub 歷史紀錄 (History) 恢復上一個版本，或者重新複製一次 AI 給的代碼再試一次。

狀況 2：出現紅色錯誤「系統未連接」或「資料庫未建立」？

解法：請檢查 Firebase 控制台，確保 Firestore 資料庫是開啟狀態，且規則 (Rules) 設定正確。通常不需要動到這裡，除非有人去動了 Firebase 設定。

狀況 3：iPad 上點擊輸入框鍵盤很難用？

解法：系統已針對 iPad 優化。點擊「數量」會跳數字鍵盤，點擊「搜尋」會跳文字鍵盤。如果跑掉，請 AI 檢查「inputmode」設定。

6. 重要金鑰 (絕對不能外流)

在 index.html 的程式碼中，有一段 const firebaseConfig = { ... }。

這是連接資料庫的鑰匙。

請AI修改程式碼時，請它務必保留這段配置，不要刪除或更改 appId。

只要 appId (pos-pro-v2-main) 不變，您的歷史營收資料就永遠都在。
