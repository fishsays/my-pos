SILK POS 系統 - 交接與維護手冊

歡迎接手 SILK POS 系統！這是一套專為 iPad 設計的輕量化銷售管理系統。

這份手冊是特別為沒有程式背景的管理者設計的。本系統的維護核心理念是：「複製程式碼給 AI，告訴它你想改什麼，然後把 AI 給你的新程式碼貼回去。」

1. 系統架構概念 (必讀)

這個系統由兩個部分組成，請想像成「店面」與「倉庫」：

店面 (GitHub Pages)：就是您在 iPad 上看到的畫面。

檔案位置：GitHub 上的 index.html。

所有功能的修改（改顏色、加按鈕、修流程）都是改這裡。

倉庫 (Firebase)：雲端資料庫，負責記住所有的商品、訂單和庫存。

資料是獨立安全的。 就算您把 index.html 改壞了，只要 Firebase 設定沒動，客人的訂單資料都不會消失。

2. 如何使用系統 (日常操作)

本系統分為四個主要頁面：

銷售 (Sales)：

點選商品或掃描條碼加入購物車。

標籤篩選：可多選標籤，系統僅會顯示「同時符合」所有選中標籤的商品。

商品套組：切換分頁可一鍵加入常用組合（如：10項套組）。

結帳：點擊右側結帳，可選擇日期、人員、付款方式與折扣。

庫存 (Inventory)：

快速進貨/盤點：直接點擊商品進行庫存調整。

進貨單：滑動列表時側邊欄會自動收合，點擊後展開，確認後可批次入庫（不產生金額）。

數量輸入：點擊數字可直接呼叫平板鍵盤輸入大額數量。

報表 (History)：

查看所有交易紀錄。

編輯：可修改「數量」、「折扣」、「人員」與「日期」，系統會自動重新計算總額。

刪除：可永久移除錯誤的紀錄。

匯出：可將資料下載為 Excel 檔。

設定 (Settings)：

新增/修改/刪除商品、套組、人員、付款方式、折扣與標籤。

支援拖曳排序。

3. 如何修改與更新功能 (AI 維護流程)

如果您需要修改功能，請嚴格遵守以下步驟：

步驟 A：取得現有程式碼

進入本專案的 GitHub 儲存庫。

點擊 index.html 檔案。

點擊右上角的 「複製圖示 (Copy raw file)」。

步驟 B：與 AI 溝通

將複製的程式碼貼給 AI（ChatGPT, Claude 或 Gemini）。

輸入指令：例如「請幫我把購物車的背景改成淺藍色」。

接收新代碼：複製 AI 生成的完整新程式碼。

步驟 C：更新上線

回到 GitHub 的 index.html 點擊 「鉛筆圖示 (Edit)」。

全選 (Ctrl+A) 舊代碼並 刪除 (Delete)。

貼上 (Ctrl+V) 新代碼並點擊 「Commit changes」。

4. 圖片上傳教學

建議將圖片上傳至 GitHub 的 images 資料夾或使用免費圖床。

取得 「直接連結 (Direct Link)」（結尾須為 .jpg 或 .png）。

進入系統 「設定」 -> 編輯商品 -> 將網址貼入 「圖片連結」 欄位。

5. 重要金鑰 (絕對不能刪除)

在 index.html 中，請務必保留 firebaseConfig 配置與 appId: 'pos-pro-v2-main'。這是連結資料庫的唯一憑證，一旦更動，資料將無法讀取。
